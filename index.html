// flujo del proyecto

  • Dropzone compacta: panel lateral inteligente con zona de arrastrar y soltar que reacciona visualmente y se oculta tras procesar el archivo en memoria local con la FileReader API.
  • Chat con inyección de contexto: el contenido del documento se fusiona con la pregunta del usuario para dar respuestas precisas basadas estrictamente en el texto.
  • Sugerencias proactivas: al subir el archivo, Gemini analiza el texto y devuelve 3 preguntas clave como pastillas interactivas en el chat.
  • Multi-idioma dinámico: la interfaz se traduce al instante a 4 idiomas y, mediante Prompt Engineering, la IA traduce el documento y responde de forma nativa en el idioma elegido por el usuario.

// portfolio v1 / build process

Cómo se construyó este portfolio

5 pasos, desde la terminal hasta producción.

  1. // 01

    Instalar opencode y abrirlo en Antigravity IDE

    opencode es un agente de IA de código abierto que se ejecuta directamente desde la terminal. La instalación es un único comando y, una vez listo, basta con lanzarlo desde la terminal integrada de Antigravity IDE para tenerlo trabajando dentro del propio editor. 

    $curl -fsSL https://opencode.ai/install | bash
  2. // 02

    Diseñar los prompts con Claude

    Antes de escribir una sola línea de código, utilicé Claude (claude.ai) para redactar prompts precisos y detallados que indicasen a opencode qué construir exactamente: diseño, secciones, efectos, paleta de colores y tipografía. La clave fue dejar muy poco margen a la improvisación: cuanto más específico es el prompt, más predecible y coherente es el resultado del agente.

  3. // 03

    Construir la web con opencode

    Con los prompts preparados, opencode generó el index.html completo con todo el CSS y el JavaScript embebidos en el mismo archivo. Un único fichero, sin frameworks, sin dependencias y sin paso de build: estética dark terminal con glassmorphism, cursor personalizado, partículas animadas, typewriter, cards con efecto tilt 3D y formulario de contacto integrado.

  4. // 04

    Crear el repositorio en GitHub

    El proyecto se subió a GitHub usando los comandos de git que el propio opencode ejecutó desde la terminal: init, add, commit y push. Tener el repo público sirve además como historial vivo de cada iteración del agente.

  5. // 05

    Desplegar en Vercel

    Por último, conecté el repositorio de GitHub con Vercel. A partir de ese momento, cada push a la rama main dispara un despliegue automático y actualiza la web en producción en cuestión de segundos, sin tener que tocar ningún panel.
